# Tier 3: 规模化运营执行指南

> **目标**：支撑大量卖家的日常运营需求，提供数据驱动的经营工具。
> **触发词**："规模化"、"Tier 3"、"PG-301"~"PG-305"
> **前置条件**：Tier 0 + Tier 1 全部完成

## PG-301: 高级分析

### 目标

为卖家提供数据驱动的经营决策支持。

### 功能模块

1. **销售分析**
   - 销售趋势图（日/周/月）
   - 收入/订单数/客单价
   - 币种分布

2. **商品表现**
   - 各商品销量排名
   - 浏览量 vs 转化率
   - 库存周转

3. **客户分析**
   - 新客 vs 回头客
   - 地域分布
   - 购买频次

4. **流量来源**
   - 直接访问 / 搜索 / 社交 / 外部链接
   - 需要前端事件埋点基础设施

### 技术依赖

- 后端需要提供聚合统计 API
- 前端使用 recharts（已安装）绘制图表
- 事件埋点（可选，Tier 3 晚期或 Tier 4）

### 实施建议

1. 先用现有订单数据做基础统计（销售额、订单数）
2. 后续接入事件系统做转化漏斗和流量分析

---

## PG-302: 客户管理

### 功能

- 买家列表（从历史订单提取）
- 每个买家的购买历史、总消费
- 标签/分组管理
- 快捷发消息

### 实施建议

MVP：从订单数据聚合买家信息，无需独立 CRM 后端。

---

## PG-303: 营销工具

### 功能

- 限时折扣（设定开始/结束时间）
- 满减活动（满 X 减 Y）
- 折扣码活动（批量生成）
- 活动管理面板

### 技术依赖

需要后端支持活动/促销规则引擎。当前优先级低于 Tier 0-2。

---

## PG-304: 多店铺管理

### 场景

一个卖家账号管理多个品牌/店铺（类似 Shopify 的多 store）。

### 依赖

SaaS 多租户架构已就绪（Phase F），但前端需要店铺切换器。

## PG-305: 插件系统（概念阶段）

### 设想

- 定义扩展点（hook points）
