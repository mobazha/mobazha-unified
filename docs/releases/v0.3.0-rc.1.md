# Mobazha Unified v0.3.0-rc.1

Status: release candidate. This version has not been tagged or published.

## Highlights

- One shared frontend `main` branch for Community Edition and other Mobazha backend deployments.
- Backend-authoritative, versioned runtime capability projection.
- Vite and Next.js entry points with the same capability and provider boundaries.
- Community checkout support for BTC, BCH, and LTC when advertised by the connected backend.
- Browser wallet integration limited to standards exposed by the browser; connector-specific SDKs are not bundled in the public core.

## Compatibility

- Node.js 20 or newer and pnpm 9.15.4 are required for the supported source build.
- The matching node release candidate is Mobazha Community Edition v0.3.0-rc.1.
- Optional UI paths remain fail-closed until a valid runtime capability response is available.
- ZEC is intentionally outside this candidate while its production settlement path is completed.

## Verification

Before tagging this candidate, release maintainers must run:

```bash
corepack pnpm@9.15.4 install --frozen-lockfile
corepack pnpm@9.15.4 lint
corepack pnpm@9.15.4 typecheck
corepack pnpm@9.15.4 --filter @mobazha/core test:unit
corepack pnpm@9.15.4 --filter @mobazha/web build
corepack pnpm@9.15.4 --filter @mobazha/web build:next
corepack pnpm@9.15.4 audit --prod --audit-level=high
```

The final release must also include production-bundle inspection, desktop and mobile browser review, dependency and secret scans, checksums, provenance, and the exact source commit used for every artifact.

## Known limitations

- This release candidate is not a stable production release.
- Runtime capability negotiation requires a compatible Mobazha backend.
- Fiat pricing and localization remain available, while bundled fiat payment providers are not enabled in the Community Edition runtime.

## Security

Do not report vulnerabilities in a public issue. Follow the private reporting process in [`SECURITY.md`](../../SECURITY.md).
